# PRS 루브릭 추적성 절차

논문이 제안한 통합 공시 지침을 신뢰할 수 있는 채점 도구로 바꾸는 유일한 방법은, 모든 루브릭 항목이 논문의 어느 문장·표에서 왔는지 되짚어갈 수 있게 만드는 것이다. 추적이 끊기는 순간 이 지표는 "회계 연구를 반영한 지표"가 아니라 "AI가 임의로 만든 체크리스트"가 된다.

## 1. 원천 데이터

논문(최아리 외, 2026)의 19개 항목 전체 목록과 원문 인용은 [references/paper-items.md](references/paper-items.md)에 정리되어 있다. 항목을 추출할 때마다 이 파일을 먼저 확인한다 — 논문 원문을 매번 처음부터 다시 읽을 필요가 없도록 이미 정리해 두었다.

## 2. rubric.yaml 항목 작성 절차

각 항목마다 다음 순서로 채운다:

1. `source_table`/`source_item_id` — references/paper-items.md에서 그대로 가져온다. 절대 새로 번호를 매기지 않는다.
2. `criteria` (0/1/2점) — 논문 본문의 서술(예: 5.2.1절 "항목7~11 설명")을 근거로 판정기준을 문장화한다. 논문이 직접 0/1/2 척도를 제시하지 않으므로, 표10의 ○(충분공시)/△(부분공시)/✕(미공시) 3단계 실증 채점을 참고하되 — **이 척도는 PRSTI가 새로 설계하는 것**이지 논문에서 가져온 것이 아님을 `docs/scoring-methodology.md`에 명시한다.
3. `accepted_example` — 논문 표4(약정에 대한 설명 예시), 표9(모범안), 부록 사례 중 실제로 그 항목을 잘 공시한 사례가 있으면 원문 그대로 인용한다. 없으면 `null`로 두고 지어내지 않는다.
4. `counter_example` — 논문 표10에서 해당 항목이 ✕(미공시)로 표시된 기업 사례를 인용한다. 표10은 12개 필수항목 전체에 대해 10개 기업의 공시 여부를 이미 채점해 두었으므로 반례 재료가 풍부하다.
5. `proposed_weight` — 사용자 승인 전에는 항상 `null`. 논문은 항목별 배점을 제시하지 않았으므로(필수/조건부필수/권장이라는 "의무 수준" 분류만 제시), 배점 설계는 PRSTI의 몫이며 회계적 판단이 필요해 임의로 정하지 않는다.

## 3. 조건부 필수항목의 N/A 처리

논문 5.2.1절은 "본 PRS 계약은 가격보전조건이 없으며, 순수 기초자산 가격 변동에 따른 차액 정산 구조"라고 명시하는 것도 유효한 공시로 인정한다. 즉 조건부 필수항목은 세 가지 상태를 가진다:

- **조건 해당 + 공시함** → 0/1/2점 정상 채점
- **조건 해당 + 미공시** → 0점 (진짜 누락)
- **조건 비해당임을 명시적으로 밝힘** → 만점 또는 분모 제외 (미정 — 이건 배점과 마찬가지로 사용자 판단이 필요하다. 두 옵션을 `_workspace/rubric_gaps.md`에 질문으로 남긴다.)

이 세 상태를 rubric.yaml의 N/A 규칙 필드로 구분해서 저장한다. "조건 비해당"과 "미공시"를 같은 0점으로 처리하면, 가격보전조건이 없는 자본성 거래가 오히려 감점당하는 왜곡이 생긴다.

## 4. 논문 미출처 항목 탐지 절차

기획초안의 5영역(거래식별성/계약구조/회계처리투명성/위험정보/정량성과비교가능성) 구조를 그대로 쓰려는 유혹이 있지만, 이 5영역은 논문 표6~8의 19개 항목을 사후적으로 재배치한 것이라 일부 하위 항목이 논문에 직접 없는 경우가 있다. 항목을 추가하기 전에 항상 다음을 확인한다:

1. references/paper-items.md의 19개 항목 목록에 정확히 있는가?
2. 없다면 논문 본문(3~4장)에서 "문제로 지적"되었을 뿐 "채점 항목으로 제안"되지는 않았는가? (예: "PRS 명칭"은 4.2절에서 기업마다 다르다는 *문제*로 지적되지만, 표6~8에는 채점 항목으로 없다.)
3. 2번에 해당하면 항목을 만들지 말고 `_workspace/rubric_gaps.md`에 "논문 미출처 항목 후보"로 기록하고 사용자에게 포함 여부를 묻는다.

이미 발견된 미출처 후보 3개(참고용, 재확인 없이 그대로 채택하지 말 것):
- **PRS 명칭** — 4.2절에서 문제로만 언급, 표6~8에 채점 항목 없음
- **공시 위치** — 5.2.3절에서 "위치 표준화"를 구조적 권고로 제시하지만, 기업별 채점 항목이 아니라 지침 자체의 설계 원칙
- **진성매각 판단** — 3.2.1절의 "위험과 보상의 이전"·금융자산 제거요건 논의와 연결되나, 표6 항목6(회계처리 근거)의 하위 판단이지 독립 항목이 아님

## 5. 기준기업 선정

논문 표10은 이미 10개 기업 × 12개 필수항목에 대해 ○/△/✕로 반정량 채점을 완료했다(공시율: 두산로보틱스 58%, 금호건설 50%, 한화솔루션·SK이노베이션·롯데케미칼·CJ ENM 46%, 효성화학 38%, 넷마블 38%, LG화학 33%, 에코프로 29%). 이 10개 기업은 Phase 0 수작업 채점의 출발점이자, 나중에 scoring-engine-dev가 회귀 테스트에 쓸 "정답지" 후보다. 임의로 다른 기업을 기준기업으로 고르지 말고, 이 목록에서 사용자와 함께 3~5곳을 선택한다.
