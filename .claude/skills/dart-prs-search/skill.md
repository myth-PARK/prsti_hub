# DART PRS 검색 절차

PRS는 기업마다 명칭과 공시 위치가 달라(논문 4.2절), 단일 키워드 검색으로는 놓친다. 이 스킬은 논문이 실제로 확인한 표현 차이를 동의어 사전으로 정리해 검색 재현율을 높인다.

## 1. 왜 동의어 사전이 필요한가

논문 표4가 보여주는 실제 표현 차이:
- 한화솔루션: "매매수익스왑(Price Return Swap)" — PRS라는 용어를 직접 씀
- SK이노베이션: PRS라는 용어 없이 "매수자의 기초자산 매각시 매각금액과 정산금액과의 차액을 수수함"이라고만 서술
- 롯데케미칼: "PRS 계약"이라고 명명

"PRS"만 검색하면 SK이노베이션 유형의 공시를 놓친다.

## 2. 검색 키워드 사전

**직접 명칭**: PRS, Price Return Swap, 주가수익스와프, 매매수익스왑, 가격수익스왑

**공시서식상 관련 결정 유형**(PRS는 이 서식들과 결합되어 공시됨):
- 타법인 주식 및 출자증권 처분결정
- 유상증자결정 (제3자배정)
- 주요사항보고서 중 위 두 유형

**구조 서술형 표현**(명칭 없이 구조만 서술하는 경우 대비):
- "기초자산" + "정산금액" + "차액"
- "만기" + "재매입" 또는 "재매수"
- "목표 수익률" 또는 "IRR" + "보장"
- "우발부채 및 약정사항" (한화솔루션처럼 이 주석 항목에 숨어있는 경우)

## 3. 검색 대상 문서

1. 사업보고서 / 반기보고서 / 분기보고서 — "Ⅲ. 재무에 관한 사항 > 재무제표 주석 > 파생상품 거래" 섹션 우선 확인(논문 5.2.3절이 권고하는 표준 위치)
2. 주요사항보고서 — 타법인 주식 및 출자증권 처분결정, 유상증자결정
3. 신종자본증권 관련 주석(파생 계약과 별도로 자본성 조달만 기재된 경우 대조용)

논문 4.2절 관찰: "분기 보고서에는 대체로 상세하지 않았으며, 더 구체적인 내용은 이후의 반기보고서나 사업보고서에서 확인되었다." — 분기보고서만 보고 "공시 없음"으로 결론내지 않는다. 후속 반기/사업보고서를 반드시 확인한다.

## 4. 소량 타겟 조회 원칙

이 스킬은 사용자가 지정한 **특정 기업 + 특정 연도**에 대한 조회용이다. 다음은 반드시 사전 승인이 필요하다(원 프롬프트의 "DART 대량 수집 금지" 제약):

- 전체 상장사 순회 검색
- 특정 업종 전체 기업 일괄 조회
- 자동화된 정기 크롤링/스케줄 수집

승인 없이 이런 대량 작업을 시작하지 않는다. 필요하다고 판단되면 먼저 사용자에게 왜 필요한지, 범위가 얼마나 되는지 알리고 승인을 받는다.

## 5. 출처 메타데이터 기록

찾은 모든 문단은 다음을 함께 기록해야 재현 가능하다:
- 문서명, 접수번호(rcept_no), 공시일자
- 문서 내 위치(섹션명, 가능하면 페이지)
- 조회 시점(retrieved_at) — DART는 정정공시로 갱신되므로 언제 봤는지가 중요하다

`dart-researcher` 에이전트 정의의 JSON 스키마를 그대로 따른다.
